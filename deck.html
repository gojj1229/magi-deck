하드웨어와 모델은 폭발 성장 중인데,
정작 사용자의 '진짜 데이터'를 다루는 인프라가 없습니다.

서비스 프로바이더(노션·MS·구글)는 자기 도메인 안에서만 작동. 사용자 입장에서는 "내가 매일 쓰는 도구들에 흩어진 정보"가 통합되지 않습니다. AI 에이전트 시대의 진짜 병목은 모델이 아니라 — 사용자 단의 큐레이션된 개인 지식 데이터입니다.

3

그래서, 우리 위키는
스치는 정보를 자동 지식화 하고 관리 하는
데이터 서비스를 제공합니다.

대화·문서·영상 — 매일 흘러가는 정보를 AI가 자동으로 읽고, 정리하고, 관리합니다. 위키는 스스로 발전합니다.

3-1 · 서비스 도식

한 번 쓰면, 위키가 스스로 자랍니다.

사용자는 그냥 평소처럼 LLM과 대화하기만 하면, 위키가 자동으로 자라납니다.

3-2 · 능동형 지식 인프라

단순히 "저장하는 DB"가 아니라,
스스로 큐레이션하고, 갭을 메우고,
지식 스파크를 발화하는 능동형 지식 인프라.

⏵
쓰던 도구 그대로 —
그저, 더 똑똑해집니다.
노션·Claude·옵시디언 그대로. magi-wiki MCP가 자동 연결되어 답이 깊어집니다. 인프라 운영은 우리가, 사용자는 평소대로.
↑
쓸수록 똑똑해집니다.
9개월 사용 = 1,766p 누적 (자체 검증). 새 메모마다 기존 자료와 자동 연결되어 — 이자에 이자가 붙듯, 시간이 곧 자산이 됩니다. 1년 후 = 답변 풍부도 10×, 2년 후 = 50×.
⊕
AI 에이전트 최적화된
퍼스널 데이터셋을 구축합니다.
에이전트가 가장 잘 읽도록 자동 큐레이션된 내 전용 데이터. 비공개 격리 + 클라우드 LLM 자동 거부 + 데이터 주권 — 내 위키 = 내 평생 자산.
⇄
다른 LLM 갈아타도 위키 그대로
Claude → ChatGPT → Gemini로 바꿔도 내 위키는 그대로 따라옵니다. LLM 락인 0.
⊛
자동 정리 — 외톨이 0%
에이전트가 자동으로 관계망을 잡아줍니다. 다른 도구는 70%+ 외톨이, 우리는 0%.
⌘
키워드 X, 의미로 연결
"비슷한 주제 다 가져와"가 가능합니다. 단어가 달라도 의미가 통하면 자동 연결.
AKI — AGENTIC KNOWLEDGE INFRASTRUCTURE · 데이터베이스의 제3세대
세대 패러다임 대표 행동 주체
1 Passive RDBMS, NoSQL (Mongo / Redis / Neo4j) 사람이 넣고 꺼냄 (CRUD)
2 Retrievable Vector DB + RAG AI가 꺼내 씀 (read-mostly)
3 Agentic magi-wiki (AKI) DB가 스스로 쓰고 · 연결 · 발화
